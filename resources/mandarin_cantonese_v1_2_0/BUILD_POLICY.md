# 数据构建与去重规范 v1.2.0

## 固定短语晋升为模板

满足以下条件时，应从 Phrases 迁移到 Templates：

1. 至少三个句子共享稳定源/目标骨架；
2. 变化部分可以归入明确 slot type；
3. 目标语序和语气在组合中稳定；
4. 组合不存在明显主语角色或时间逻辑冲突。

## 禁止项

- 禁止将模板的时间 × 人物 × 动作笛卡尔积写入 Phrases；
- 禁止在 Lexicon 或固定 Phrases 中保存仅简繁/异体转换条目；
- 禁止使用无类型的开放 ACTION 槽位处理服务、医疗等高风险场景；
- 禁止把测试句计作翻译单元。

## 允许的身份结构模板

如果模板源目标骨架相同，但满足以下至少一个条件，可以保留：

- 槽位包含真实词汇转换；
- 模板提供必要的量词、角色或意图约束；
- 模板声明体貌或语序变换；
- 模板用于限制合法组合。

此类模板必须标记 `identity_structure=true`，并且至少引用一个 typed slot 或 transform。

## 发布校验

构建必须验证：

- Phrases 中没有 `generated_candidate` 笛卡尔积行；
- 所有模板引用的 slot type 存在；
- closed-list slot type 至少有一个值；
- Lexicon 和固定 Phrases 没有显式同词直通映射；
- 所有迁移短句都能追溯到目标模板；
- Templates 没有结构重复。
