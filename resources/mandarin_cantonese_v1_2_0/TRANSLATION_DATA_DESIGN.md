# 普通话口语到粤语口语离线翻译数据设计 v1.2.0

## 1. 重构目标

旧版将大量句型和槽位组合预先展开为固定短句，例如：

- 我想在中午寄存行李
- 我想在下午寄存行李
- 我想在中午打扫房间
- 我想在下午打扫房间

这类数据会快速增加条目数，却不能有效增加句法和意图覆盖。v1.2.0 将其改为：

```text
固定短语覆盖
→ 类型化模板
→ 槽位识别与翻译
→ 通用语法规则
→ 词汇后备
```

## 2. 五层数据职责

### Phrases

只保存不可可靠组合推导的固定表达、惯用语和人工例外。旧版
6,477 条受控生成短句已从运行数据中移除，保留
369 条人工固定表达。

### Templates

保存完整意图、句法结构、语序和槽位关系。当前包括：

- 人工模板：90
- 场景框架模板：152
- 压缩后的通用组合模板：30

示例：

```text
源：我想在{schedule_time:schedule_time}{scheduled_service:scheduled_service}。
粤：我想喺{schedule_time}{scheduled_service}。
```

### Slot Types

定义槽位的语义类型和约束，例如：

- `schedule_time`
- `scheduled_service`
- `hotel_room_service`
- `route_place`
- `ingestible_medicine`

通用的 `ACTION` 槽位过宽时，必须拆成更窄的语义槽位，防止产生不自然组合。

### Slot Values

保存模板识别所需的词和值。它与 Lexicon 的职责不同：

- `mapped_value_metadata`：槽位源值和目标值不同；
- `same_form_metadata`：源目标相同，但仍用于识别物品、地点、动作等语义类型。

这些元数据不计入“翻译映射条目”，也不参与普通词汇最长匹配。

### Rules 与 Lexicon

- Rules：否定、体貌、比较、介词、语序和语气等跨模板规则；
- Lexicon：真正发生词汇变化、且不能仅靠简繁转换完成的映射。

## 3. Templates 与 Rules 的边界

模板负责完整语义框架：

```text
我想让你们在{TIME}{SERVICE}
→ 我想你哋喺{TIME}幫我{SERVICE}
```

规则负责可跨句型复用的局部变化：

```text
在 + TIME/PLACE → 喺 + TIME/PLACE
还没有 + VP → 仲未 + VP
正在 + VP → VP + 緊
```

不能用一个无限制规则替代具有主语角色、服务对象和礼貌语气的完整模板。

## 4. 运行时匹配

1. 简繁和标点规范化；
2. 固定短语精确覆盖；
3. 场景与意图模板匹配；
4. 按 slot type 识别槽位；
5. 翻译槽位并执行模板变换；
6. 应用局部语法规则；
7. 使用 Lexicon 处理剩余词汇；
8. 对候选结果评分。

## 5. 可选槽位

通用模板不再展开“我/你/我们 × 今天/明天/现在”等排列。一个模板可以提供多组对应模式：

```json
{
  "source_patterns": [
    "{person:person}想{vp:VP}。",
    "{time:generic_time_prefix}{person:person}想{vp:VP}。"
  ],
  "target_patterns": [
    "{person}想{vp}。",
    "{time}{person}想{vp}。"
  ]
}
```

## 6. 体貌变换

进行体和完成体不能简单把“紧/咗”放在整个动词短语末尾。模板通过
`transform_pipeline` 声明：

```json
{"type":"aspect_after_head","slot":"daily_action","marker":"緊"}
```

例如：

```text
正在打扫房间 → 執緊房
已经坐地铁了 → 已經搭咗港鐵
```

## 7. 覆盖衡量

不再以展开后的短句数量衡量覆盖。核心指标改为：

- 固定例外数量；
- 独立意图数量；
- 模板数量；
- 类型化槽位数量；
- 槽位值数量；
- 合法模板组合容量；
- 回归测试数量；
- 低置信度率。

当前152个场景模板的理论组合容量为 16,515，高于旧版实际保存的
6,477 条生成短句，同时数据体积和重复显著下降。

## 8. 数据扩充规范

1. 先判断是否为固定表达；
2. 多个句子只替换时间、地点、动作、物品或人物时，优先建立模板；
3. 槽位必须带语义类型，禁止无限制 `ACTION`；
4. 目标语序或语气不稳定的组合不得自动合并；
5. 异常项保留为 Phrase，且优先级高于模板；
6. 不预生成模板的笛卡尔积；
7. 测试可以保存具体句子，但不计为翻译单元。
